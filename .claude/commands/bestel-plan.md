# /bestel-plan

Use this command before non-trivial implementation.

Task: $ARGUMENTS

Follow this process:

1. Classify the subsystem.
2. Read the relevant architecture doc and target files.
3. Identify the smallest viable change.
4. List files likely to change.
5. List tests/checks to run.
6. Name the specialist agent that should own the implementation.

Return a short plan. Do not implement yet unless explicitly asked.
